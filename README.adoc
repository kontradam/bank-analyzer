== Problem biznesowy

Aplikacja konsolowa oraz graficzna (JavaFX) służąca do analizy wyciągów bankowych w formacie CSV pobieranych ze stron banków, z automatycznym przypisywaniem transakcji do kategorii i generowaniem raportów statystycznych.
Projekt powstał z osobistej potrzeby kontroli finansów prowadzonych na wielu kontach bankowych. Aplikacja automatyzuje proces segregowania wydatków oraz tworzenia zestawień finansowych, umożliwiając szybką i precyzyjną analizę nawyków wydatkowych.
Mechanizm przyporządkowywania kategorii opiera się na analizie słów kluczowych występujących w tytułach przelewów oraz nazwach odbiorców.

Do aplikacji dołączono dwa pliki CSV z fikcyjnymi danymi (bank_data) przeznaczonymi do testów manualnych.

== Funkcjonalności

* Multi-file processing - jednoczesne wczytywanie wielu plików CSV
* Automatyczna kategoryzacja - inteligentne przypisywanie transakcji do 10 kategorii
* Analiza wydatków - sumy i rankingi według kategorii
* Zestawienie miesięczne - przychody, wydatki i bilans dla każdego miesiąca
* TOP 10 wydatków - ranking najdroższych transakcji
* Export raportu - zapis pełnej analizy do pliku TXT
* Interfejs graficzny - wykresy wydatków i tabela transakcji (JavaFX)

* Java 17

== Instalacja i uruchomienie

=== Wymagania

Java 17 lub nowsza (JDK, nie JRE)

Sprawdzenie wersji:
----
java -version
----

=== Instalacja

Sklonuj repozytorium:
----
git clone https://github.com/kontradam/bank-analyzer.git
cd bank-analyzer
----

Zbuduj projekt:
----
gradlew.bat build
----

=== Uruchomienie

Interfejs graficzny (GUI):
----
gradlew.bat run
----

Aplikacja konsolowa:
----
java -jar build/libs/bank-analyzer-1.0.0.jar plik1.csv plik2.csv
----

=== Testy

----
gradlew.bat test
----

== Format pliku CSV

Aplikacja obsługuje format eksportu z PKO BP.

Format innych banków (mBank, ING, Santander) wymaga dostosowania parsera w klasie `CsvParserService`.
